В цей блок було винесено в основному параметри ширини та відображення різних елементів, оскільки інші атрибути, наприклад колір, шрифт і т.д. збігаються для всіх версій сайту і немає сенсу їх дублювати в блоках медіа-апитів.

Для реалізації мобільної версії сайту було внесені найбільші зміни в початковий макет.

Цього разу також був використаний медіа-запит @media (max-width:700px), проте з атрибутом max-width, що означає, що вказані нижче стилі будуть використані для екранів з шириною манше 700 пікселів. Отже ций макет буде використовуватись для телефонів або ж в випадку, коли розмір вікна сайту доволі вузьке.

В мобільній версії сайту головна панель навігації відображається вже не по рядкам, а стовпцем, щоб кнопками було зручно користуватись на маленьких екранах, а текст мав достатньо місця для розміщення. Footer тепер також розміщується вертикально, а мапа прибрана. З header'а була прибрана вся додаткова інформація та перенесена в footer, залишивши зверху тільки назву ресторану. Також в основній частині сайту було прибрана додаткова частина, що в інших версіях відображається зліва від основної.

Медіа-запити - це правила CSS, що дозволяють керувати стилями елементів залежно від значень технічних параметрів (висота та ширина браузера, роздільна здатність сторінки, типу пристрою, його орієнтації, щільності пікселів тощо).

Медіа-запити описуються так: 

@media media-type {
  /* CSS-правила */
}

Де media-type - медіа-тип, який визначає, на які пристрої будуть застосовуватися CSS-правила.

Медіа-типи можна розділити на кілька категорій:

За розміром пристрою:

  • all - всі пристрої
  • screen - дисплеї
  • print - принтери
  • handheld - портативні пристрої

За орієнтацією пристрою:
  • portrait - портретна орієнтація
  • landscape - альбомна орієнтація

За типом пристрою:
  • (device-type: desktop) - комп'ютери
  • (device-type: mobile) - мобільні пристрої
  • (device-type: tablet) - планшети

За роздільною здатністю пристрою:
  • (min-width: 400px) - ширина пристрою не менше 400 пікселів
  • (max-width: 800px) - ширина пристрою не більше 800 пікселів

За щільністю пікселів пристрою:
  • (min-device-pixel-ratio: 2) - щільність пікселів пристрою не менше 2
  • (max-device-pixel-ratio: 3) - щільність пікселів пристрою не більше 3

Медіа-функції - це функції, які використовуються для визначення значень технічних параметрів пристрою.

Деякі з найпоширеніших медіа-функцій:

  • min-width() - мінімальна ширина пристрою
  • max-width() - максимальна ширина пристрою
  • min-height() - мінімальна висота пристрою
  • max-height() - максимальна висота пристрою
  • orientation() - орієнтація пристрою
  • device-type() - тип пристрою
  • device-pixel-ratio() - щільність пікселів пристрою

Метатег viewport - це метатег HTML, який дозволяє вказати браузеру, як він повинен відображати сторінку в залежності від розміру пристрою.

Метатег viewport описується так: 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Де:
  • width=device-width - вказує браузеру відображати сторінку так, щоб її ширина відповідала ширині пристрою.
  • initial-scale=1.0 - вказує браузеру використовувати початковий масштаб 1.0.

Під час цієї лаборатоної роботи ми навчилися створювати адаптивні верстки для різних пристроїв, а саме:

  • Десктопна версія
  • Планшетна версія
  • Мобільна версія

Ми використали медіа-запити та медіа-функції для адаптації нашого сайту під різні типи пристроїв, щоб його використання було зручне для всіх відвідувачів.

Також ми впевнились в актуальності статегії Mobile First, оскільки для адаптації декстопної версії сайту під мобільні телефони довелося майже повністю змінювати верстку сайту, чого не відбулось би, якби з самого початку розробки сайту була використана ця стратегія.
